## Features

- **Balance Tracking** вЂ” Monitor account balances over time with trend charts
- **Transactions** вЂ” Log income and expenses with categories, filters, and recurring support
- **Budget Management** вЂ” Set monthly budgets per category with visual progress bars
- **Savings Goals** вЂ” Create goals with deadlines, track contributions, see required daily/weekly rates
- **Debt Tracking** вЂ” Track debts, interest rates, and payment schedules
- **Net Worth** вЂ” Aggregate view of assets vs liabilities over time
- **Financial Forecasting** вЂ” Project future balances based on spending patterns
- **Charts** вЂ” Balance trend line chart, spending breakdown donut chart (Recharts)
- **AI Insights** вЂ” Spending optimization, 60-day profile builder, financial chat assistant
- **Reports** вЂ” Auto-generated weekly and monthly reports with AI narrative
- **Data Import** вЂ” Import transactions from CSV/bank exports
- **Dark/Light Mode** вЂ” Toggle between light, dark, or system preference
- **Configurable Currency** вЂ” USD, EUR, GBP, and more
- **HTTPS by Default** вЂ” Locally-trusted TLS via mkcert + Nginx reverse proxy
- **Multi-User** вЂ” Individual accounts with isolated data
- **PWA** вЂ” Installable as a progressive web app with offline caching

## Quick Start (Docker)

The fastest way to run FinTrack. Only requires Docker вЂ” no Node.js or dependencies to install.

### 1. Install Docker (if you don't have it)

**Mac:**
```bash
brew install --cask docker
open /Applications/Docker.app
```

**Linux (Ubuntu/Debian):**
```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
newgrp docker
```

### 2. Clone and Run

```bash
git clone https://github.com/b-3llum/finance-tracker.git
cd finance-tracker
docker compose up --build -d
```

The app is now running on **http://localhost:3000** (HTTP). Register an account and start tracking.

Your data is stored in a Docker volume and persists across restarts and rebuilds.

## HTTPS Setup

For encrypted traffic (recommended), set up locally-trusted HTTPS with mkcert. This gives you a green padlock on `https://localhost` with zero browser warnings вЂ” works in Chrome, Safari, Firefox, and Edge.

### 1. Install mkcert

**Mac:**
```bash
brew install mkcert
```

**Linux:**
```bash
sudo apt install libnss3-tools
curl -JLO "https://dl.filippo.io/mkcert/latest?for=linux/amd64"
chmod +x mkcert-v*-linux-amd64
sudo mv mkcert-v*-linux-amd64 /usr/local/bin/mkcert
```

### 2. Generate Certs and Start

```bash
./setup-https.sh
docker compose up -d --build
```

Open **https://localhost** вЂ” that's it. Port 80 redirects to HTTPS automatically.

### How It Works

```
Browser в”Ђв”ЂHTTPSв”Ђв”Ђв–¶ Nginx (port 443) в”Ђв”ЂHTTPв”Ђв”Ђв–¶ Next.js (port 3000, internal)
                   TLS termination            App container (not exposed)
```

Nginx handles TLS termination using the certs generated by mkcert. The Next.js app runs on an internal Docker network and is never directly exposed.

## Manual Setup (without Docker)

### Prerequisites

- Node.js 22+ with npm
- (Optional) [Ollama](https://ollama.ai) for AI features: `ollama pull llama3`

### Install and Run

```bash
git clone https://github.com/b-3llum/finance-tracker.git
cd finance-tracker
npm install
cp .env.local.example .env.local
npm run dev
```

Open **http://localhost:3000**

## Managing the App

```bash
docker compose down          # stop
docker compose up -d         # start (data preserved)
docker compose up -d --build # rebuild after code changes
docker compose logs -f       # view logs
```

### Persistent Secrets

By default, `JWT_SECRET` and `ENCRYPTION_KEY` are auto-generated on each container start. This means sessions invalidate on restart. To keep sessions across rebuilds, set them explicitly in `docker-compose.yml`:

```yaml
environment:
  - JWT_SECRET=your-secret-here
  - ENCRYPTION_KEY=your-encryption-key-here
```

Generate a strong secret: `openssl rand -base64 32`

## AI Features

All AI features work with multiple providers. Set your provider in **Settings**.

| Feature | Description |
|---|---|
| **Spending Profile** | After 14+ days of data, generates a personality profile with habits analysis |
| **Spending Optimization** | Analyzes current month and suggests specific cuts |
| **Financial Chat** | Ask questions about your finances with streaming responses |
| **Report Insights** | AI-written narrative summaries in weekly/monthly reports |
| **Anomaly Detection** | Flags unusual spending patterns |
| **Subscription Detection** | Identifies recurring charges from transaction history |

### Enable Ollama (Local AI)

```bash
# Install Ollama
curl -fsSL https://ollama.com/install.sh | sh  # Linux
brew install ollama                              # Mac

# Pull a model
ollama pull llama3

# Update docker-compose.yml вЂ” uncomment OLLAMA_BASE_URL:
# Linux: http://172.17.0.1:11434
# Mac:   http://host.docker.internal:11434
```

Or uncomment the Ollama service in `docker-compose.yml` to run it inside Docker.

### Cloud AI Providers

In **Settings**, enter API keys for Claude (Anthropic) or OpenAI to use cloud models instead.
